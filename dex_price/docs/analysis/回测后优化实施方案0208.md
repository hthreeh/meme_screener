实现分段止损和趋势延期
根据回测报告的结论，需要将分段止损和趋势延期功能应用到实际交易代码中。

📊 背景
回测数据显示:

分段止损(-15%/-30%): 效果提升 +126%，将总 PnL 从 -2.4 变为 +0.6
趋势延期: 效果提升 +9.4%，但报告标注"可选"，因此需要做成开关控制
Proposed Changes
配置模块
[MODIFY] 
settings.py
在 
AppSettings
 类中添加新的配置字段:

staged_stop_loss_enabled: 分段止损开关 (默认 True)
trend_extension_enabled: 趋势延期开关 (默认 False, 因为报告标注"可选")
staged_stop_loss: 分段止损参数
level_1: (-0.15, 0.5) - 跌15%减仓50%
level_2: (-0.30, 1.0) - 跌30%清仓
trend_extension: 趋势延期参数
threshold
: 0.10 - 涨10%触发延期
extension_minutes: 30 - 延期30分钟
max_times: 2 - 最多延期2次
[MODIFY] 
email.json
添加新的配置节点:

json
{
  "staged_stop_loss": {
    "enabled": true,
    "level_1": {"trigger": -0.15, "sell_ratio": 0.5},
    "level_2": {"trigger": -0.30, "sell_ratio": 1.0}
  },
  "trend_extension": {
    "enabled": false,
    "threshold": 0.10,
    "extension_minutes": 30,
    "max_times": 2
  }
}
